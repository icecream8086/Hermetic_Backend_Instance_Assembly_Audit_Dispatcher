 1. SandboxTemplate 结构总览
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SandboxTemplate
│
├── 模板信息
│   ├── id: string
│   ├── name: string              ← 锁原子性基础，改名 = 新锁
│   ├── description?: string
│   ├── dependsOn?: string[]
│   ├── creatorId?: string
│   ├── createdAt / updatedAt
│   ├── visibility?: 'public'|'private'
│   ├── userGroupIds?: string[]
│   │
│   ├── singleton?: boolean       ← 单例开关, 与 instanceLimit 互斥
│   ├── instanceLimit?:           ← 数量限制, 与 singleton 互斥
│   │   ├── type: 'fixed'|'perUser'|'perSystem'
│   │   └── max: number
│   └── resourceBinding?:
│       ├── domain?: string
│       └── port?: number
│
├── container（无状态核心, tag: container, 与 containerGroup 互斥）
│   ├── region / zone / account
│   ├── restartPolicy
│   ├── containers: ContainerDef[]
│   │   ├── name / image
│   │   ├── command? / args?     ← ENTRYPOINT / CMD 覆盖
│   │   ├── env[]?               ← {name, value?, valueFrom?}
│   │   ├── ports[]?             ← {containerPort, protocol?}
│   │   └── resources?           ← {requests?, limits?{cpu,memory,gpu}}
│   └── initContainers?: ContainerDef[]
│
├── healthChecks: HealthCheckDef[]（独立）
│   ├── name / target             ← "container:name"|"init:name"
│   ├── type                      ← "liveness"|"readiness"|"startup"
│   ├── probe: ProbeSpec          ← exec/httpGet/tcpSocket
│   └── timing                    ← delay/period/timeout/thresholds
│
├── network（公共基础设施层）
│   ├── mode?: 'public'|'private'|'vpc'
│   ├── publicIp?: { allocate?, bandwidth? }
│   ├── vpc?: { id?, subnetIds[], securityGroupId? }
│   └── ← future: dns, loadBalancer
│
└── extensions（扩展功能）
    ├── storage: TemplateStorage[]
    │   ├── name / type / mountPath
    │   └── nfs? / hostPath? / emptyDir? / oss? (+ providerOverrides)
    ├── spotStrategy?
    ├── providerOverrides          ← 厂商透传
    │   ├── podman: { capAdd, privileged, networkMode, ... }
    │   ├── alibaba: { eipBandwidth, instanceType, ... }
    │   └── stub: { simulateDelayMs }
    ├── healthMaxRetries           ← 健康检查重试
    ├── autoStart
    ├── webTerminal
    └── lifecycleHooks

 2. 实例数量限制系统
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2.1 两种模式（互斥）

┌─────────────────────────────────────────────────────────────┐
│ singleton: true                                             │
│   → 自动限制为 1 个实例                                     │
│   → 开关式，不传参                                          │
│   → 等价于 instanceLimit { type: "fixed", max: 1 }          │
│   → 使用模板名 hash 作为锁 key                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ instanceLimit: { type, max }                                │
│   → type: "fixed"     — 全局 N 个                           │
│   → type: "perSystem" — 同 fixed                           │
│   → type: "perUser"   — 每用户 N 个                        │
│   → 使用模板名 hash 作为锁 key                              │
└─────────────────────────────────────────────────────────────┘

2.2 锁机制

  模板名 ──hash──→ tpl:lock:<hash> ← 原子计数器

  改名 = 新锁：对同名模板重命名后，原有容器不计入新锁。

2.3 实时计数

  apply 时扫描 sandbox:ids 索引，统计 name === 模板名 的沙箱。
  LIVE 状态计入：Pending / Scheduling / Running / Stopped / Terminated
  不计入：Failed / Deleted（允许重新创建）

  双重保障：OCC 计数器 + 实时沙箱状态扫描。

2.4 DAG 继承规则

  singleton / instanceLimit 均不被 DAG 合并。
  子模板必须显式声明自己的限制（或不声明 = 无限制）。

 3. DAG 继承规则
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

合并字段（子覆盖父）:
  container.containers        → mergeByName
  container.initContainers    → mergeByName
  healthChecks                → mergeBy(target:name)
  network                     → deep merge
  extensions                  → deep merge

不合并字段（子必须重声明）:
  singleton                   ← 独立锁
  instanceLimit               ← 独立锁
  resourceBinding             ← 独立域名绑定
  visibility                  ← 独立可见性
  id / name / dependsOn       ← 模板标识

 4. 虚拟网段系统（VirtualNetwork）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

VirtualNetwork
├── id / name / description
├── cidr: string                   ← 超网 "10.2.0.0/16"
├── subnetPrefix: number           ← 子网粒度 /24
├── securityGroupId?
├── provider / region
├── providerNetworkId?             ← 创建后回填
├── visibility: 'public'|'private'
├── creatorId?
├── userIds[] / userGroupIds[]
├── status: 'Active'|'Inactive'|'Error'
└── createdAt / updatedAt

4.1 API

  POST   /api/networks          创建（root only）
  GET    /api/networks          列出（按可见性过滤）
  GET    /api/networks/:id      详情
  PUT    /api/networks/:id      更新（root only）
  DELETE /api/networks/:id      删除（root only）

4.2 可见性规则

  ≥ root 组 + role='root'  → 绕过所有限制
  public                    → 所有认证用户可见
  private + creatorId 匹配  → 仅创建者
  private + userIds 白名单  → 指定用户
  private + userGroupIds    → 指定用户组

4.3 Route ACL

  root  → * /api/networks
  users → GET /api/networks
  wheel → * /（全路径放行）

  │
  ├─ resolveTemplate()      ← DAG 合并 container/healthChecks/network/extensions
  │                            (singleton/instanceLimit 不被合并)
  ├─ claimInstanceSlot()    ← 检查 singleton/instanceLimit + 运行时计数
  ├─ claimResourceBinding() ← 检查 domain:port 排他
  │
  ├─ applyTemplate()        ← Template → CreateSandboxInput
  │   ├─ container.containers     → input.containers
  │   ├─ healthChecks             → per-container probes
  │   ├─ network                  → input.network
  │   ├─ extensions.storage       → volumes + volumeMounts
  │   ├─ extensions.spotStrategy  → input.spotStrategy
  │   └─ extensions.providerOverrides → input.providerOverrides
  │
  ├─ svc.provision()        ← 创建沙箱实体
  │   └─ toContainerGroupInput()  → CreateContainerGroupInput
  └─ IContainerProvider.create()  ← 厂商 provider
      ├─ PodmanContainerProvider
      ├─ AlibabaEciContainerProvider
      └─ StubContainerProvider
